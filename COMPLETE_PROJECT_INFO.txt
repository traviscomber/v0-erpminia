4. BRANDBOOK - SISTEMA DE COLORES (CRÍTICO - NO HAY EXCEPCIONES)
================================================================================

REGLA DE ORO: SOLO 4 COLORES PERMITIDOS EN TODO EL PROYECTO

Color 1: PRIMARY (NARANJA - #ff6b35)
  Uso: Acciones principales, botones, iconos de acción
  Tokens: bg-primary, text-primary, border-primary, bg-primary/10
  Ejemplo: <button className="bg-primary text-primary-foreground">Acción</button>

Color 2: SECONDARY (VERDE - #00d084 aproximado)
  Uso: Éxito, completado, aprobado, estados positivos
  Tokens: bg-secondary, text-secondary, border-secondary, bg-secondary/10
  Ejemplo: <div className="bg-secondary/10 text-secondary">Completado</div>

Color 3: DESTRUCTIVE (ROJO - #ff4757)
  Uso: Errores, eliminación, estados críticos, alerta
  Tokens: bg-destructive, text-destructive, border-destructive, bg-destructive/10
  Ejemplo: <div className="bg-destructive text-destructive-foreground">Error</div>

Color 4: MUTED (GRIS OSCURO - #0f172a)
  Uso: Neutro, deshabilitado, fondo secundario
  Tokens: bg-muted, text-muted-foreground, border-muted
  Ejemplo: <span className="text-muted-foreground">Información</span>

PROHIBIDO (100% - Sin excepciones):
❌ bg-blue, bg-green, bg-red, bg-orange (raw colors)
❌ bg-blue-500, bg-green-600, bg-orange-100, etc.
❌ text-blue, text-purple, text-violet, text-cyan, etc.
❌ Gradients (from-, to-, gradient)
❌ Colores arbitrarios como #FF6B35 (usar tokens en lugar)
❌ 5+ colores en una página
❌ text-white, bg-white (usar design tokens)

OBLIGATORIO:
✅ SIEMPRE usar semantic tokens: primary, secondary, destructive, muted
✅ Leer BRANDBOOK.md ANTES de cualquier edit
✅ Validar colores con: grep -r "bg-blue\|bg-red\|bg-green" app/
✅ Nuevas páginas = copiar desde /app/dashboard/sostenibilidad/

EJEMPLOS CORRECTOS:
// ✅ CORRECTO
<div className="bg-primary/10 text-primary border border-primary/20">
<button className="bg-secondary text-secondary-foreground">
<span className="bg-destructive/5 text-destructive">

9. BUGS & FIXES
================================================================================

BUG #1 - FIXED: Sidebar Navigation Issue
Problema:    Clicking sidebar links didn't navigate to sub-modules
Symptom:     URL stayed the same, no navigation occurred
Root Cause:  Next.js Link component wrapping Button component blocked navigation
Location:    /components/layout/sidebar.tsx (lines ~358-372)
Solution:    Replaced Link wrapper with direct router.push() call
Code Change:
  ANTES:
    <Link key={item.href} href={item.href}>
      <Button onClick={() => setIsOpen(false)}>
        {item.label}
      </Button>
    </Link>
  
  DESPUÉS:
    <Button
      key={item.href}
      onClick={() => handleNavigation(item.href)}
    >
      {item.label}
    </Button>
  
  (handleNavigation: router.push(href) + setIsOpen(false))

Status:      DEPLOYED - Requires testing after hot reload
Testing:     Click on Sostenibilidad → Calendario, URL debe cambiar a /dashboard/sostenibilidad/calendario

Severity:    LOW (doesn't affect functionality)
Location:    Likely in app/layout.tsx or ThemeProvider
